# Princeton Instruments (Acton SP-2750) グレーティング/波長制御 機能拡張

Generated by Claude Code

## 背景

ユーザーがSP-2750の実機で確認した通り、`GRATING`/`GOTO`系コマンドは完了時に(echoの有無に関わらず)
`ok`を返す同期コマンドであり、この仕様を前提に中心波長・グレーティング切替は既に
`src/spectrometer_princeton.py`の`_send_command`(echoあり/なし両対応の`ok`待ちパーサ)で実装済み。

これを踏まえ、追加でリサーチした以下のコマンド群を使った機能改善を提案し、ユーザーと合意した
実装範囲を記録する:

| コマンド | 用途 |
|---|---|
| `?GRATINGS` | 全グレーティングスロットの溝本数一覧を実機から取得 |
| `MONO-STOP` | 移動中のモノクロメータを中断 |
| `NM/MIN`/`>NM`/`MONO-?DONE` | 非同期スキャン+進捗ポーリング(**今回は見送り**) |
| `TURRET`/`?TURRET`/`?TURRETS` | タレット切替(**今回は見送り**) |
| `ECHO`/`NO-ECHO` | エコー制御(**今回は見送り**) |

## 合意した実装範囲

- **A. 初回セットアップ時のグレーティング自動検出**: `spectrometerConfig.json`が存在せず、
  Configウィザードで供給元にPrinceton Instrumentsを選んだ場合、COMポート確定後に`?GRATINGS`を
  一度実機へ送ってみて、成功すればグレーティング欄(溝本数)の初期値として自動反映する。
  失敗時(未接続・パース不能)は現行通り手入力にフォールバックする。
- **B. 起動時のグレーティング整合性チェック**: 毎回の起動時、`spectrometerConfig.json`記載の
  溝本数と実機`?GRATINGS`応答を突き合わせ、食い違いがあれば警告ダイアログを出す(自動修正はしない、
  警告のみ)。
- **C. `MONO-STOP`による移動中断**: 「Spectrometer is moving」ダイアログにキャンセルボタンを追加し、
  波長移動(`GOTO`)中に限り`MONO-STOP`を送って中断できるようにする。グレーティング切替
  (`GRATING`)フェーズは中断非対応(ボタンは波長移動フェーズに入ってから有効化)。
  中断後は実機から現在位置を読み直し、UI状態を実際の位置に同期する。
- **D. ライブ進捗表示(`>NM`+`MONO-?DONE`ポーリング)**: **見送り**。`GOTO`は完了までokを返さない
  同期コマンドのため、移動中に`?NM`をポーリングするには非同期`>NM`への切替が必要になるが、これは
  実機未検証の`NM/MIN`速度特性に依存し、既に検証済みの`GOTO`(最大速度移動)の速度特性を変えて
  しまうリスクがある。ユーザーと相談の結果、今回は実装せず、実機検証の目処が立ってから
  改めて計画することにした。
- **E. `get_wavelength`/`initialize`の`_send_command`統一(最小限)**: 現状この2箇所だけが
  echo非対応の素朴な`readline()`ベース(`_send_wavelength_command`)を使っており、`_read_grating`
  等が使う堅牢な`_send_command`と実装が二重化している。動作を変えずに`_send_command`へ統一し、
  `_send_wavelength_command`を削除する。

F(タレット対応)・G(スキャン速度/`ECHO`)は保留。

## 実装ステップ

### Step E: `_send_command`への統一
**対象:** `src/spectrometer_princeton.py`

(`_send_wavelength_command("? NM")`)を、他コマンドと同じ`_send_command("?NM", ...)`に置き換え、

### Step A: `?GRATINGS`自動検出
**対象:** `src/spectrometer_princeton.py`, `src/config_wizard.py`
- `SpectrometerControllerPI.get_gratings()`: `?GRATINGS`を送り、応答行から
  `(index, grooves)`のペアを正規表現でパースして返す(`*`によるカレント選択マーク等の付随情報は
  無視、パース不能行は無視、1件も取れなければ空リスト)。**実機での応答フォーマットは未検証**
  であり、パース正規表現は実機ログを見て調整が必要になる可能性がある。
- `config_wizard.py`: `_PageGrating`にバックグラウンド検出スレッド(既存の`_PathSearchThread`と
  同じパターン)を追加。供給元がPIの場合、Paths→Gratingページ遷移時にCOMポートへ接続を試み、
  成功すれば溝本数欄を自動入力(ユーザーは編集可能)、失敗すれば現行通り手入力。

### Step B: グレーティング整合性チェック
**対象:** `src/ui.py`
起動時`self.spec_ctrl.initialize()`の直後、`get_gratings()`が使えれば(Andorでは
`hasattr`チェックでスキップ)実機値と`config["grating"]`を`index`単位で突合し、
食い違うスロットがあれば`QMessageBox.warning`で一覧表示する。設定ファイルは変更しない。

### Step C: `MONO-STOP`によるキャンセル
**対象:** `src/spectrometer_princeton.py`, `src/ui_mixins/spectrometer_control_mixin.py`
- `SpectrometerControllerPI`に`threading.Event`ベースの`request_cancel_move()`を追加。
  `_send_command`に`cancellable`引数を追加し、待機ループ中にキャンセル要求を検知したら
  `MONO-STOP`を送って残り待機時間を短縮する。`set_wavelength`のみ`cancellable=True`で呼ぶ
  (グレーティング切替は対象外)。
- `SpectrometerMoveThread`(Princeton版)に`phase_signal(str)`("grating"→"wavelength")と
  `cancelled`フラグを追加。
- 移動ダイアログにキャンセルボタンを追加(Princeton接続時のみ、`hasattr`で判定)。波長移動
  フェーズに入るまで無効化。キャンセル成功時は実機から現在波長/グレーティングを読み直して
  UIへ反映する。
- **実機での`MONO-STOP`が`GOTO`実行中に受け付けられること自体は未検証**であり、実機確認後に
  本セクションを更新する。
