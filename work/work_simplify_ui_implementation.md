## 採用する方針: Mixin パターン

このクラスはQtのシグナル/スロットと `self` 上に散らばった共有 state（ROI設定・バックグラウンドデータ・
連続測定の状態など）に強く依存しているため、単純にモジュールを分けて「別オブジェクトに切り出す」方式
（コントローラオブジェクト＋委譲）はコンストラクタとシグナル配線の書き換えが大きくなり、テストが無い
このリポジトリではリグレッションのリスクが高い。

そこで、**責務ごとにメソッド群だけを別ファイルの plain class（Mixin）に移し、`SpectrometerGUI` が
それらを多重継承する**方式を取る。

```python
class SpectrometerGUI(QMainWindow, ConfigMixin, FileIOMixin, SpectrometerControlMixin,
                       SequentialMixin, AcquisitionMixin, DisplayMixin, PressureDialogMixin):
    def __init__(self, debug=False):
        ...  # ウィジェット構築とシグナル配線はそのまま残る
```

- Mixin は `QObject` 等を継承しない素の Python class（`class ConfigMixin:` のように書く）。
- Mixin 内のメソッドは今まで通り `self.xxx` で他の属性・他Mixinのメソッドを参照してよい
  （実行時には全部同じ `SpectrometerGUI` インスタンスの属性になるため、相互参照は問題なく動く）。
- **振る舞いは一切変えない。** これは純粋なコード移動（cut & paste）であり、リファクタリングや
  バグ修正はここでは行わない。作業中に既存バグに気づいても直さず、作業報告に記載するだけに留める。
- コメント（日本語）はそのまま移動する。翻訳・削除・追記はしない。
- 新規に docstring やコメントは追加しない（CLAUDE.md の方針どおり）。
